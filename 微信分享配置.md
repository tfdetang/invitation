# 微信分享卡片配置指南

## 一、基础配置（已完成）

已在 `index.html` 中添加了以下 meta 标签：
- Open Graph 标签（微信、微博、QQ 等社交平台通用）
- Twitter Card 标签
- Schema.org 标签（微信可能读取）

## 二、重要步骤

### 1. 修改域名

请将 `index.html` 中的 `https://your-domain.com/` 替换为你的实际域名：

```html
<!-- 替换所有出现的地方 -->
<meta property="og:url" content="https://你的实际域名.com/" />
<meta property="og:image" content="https://你的实际域名.com/img/栗子.png" />
<meta name="twitter:url" content="https://你的实际域名.com/" />
<meta name="twitter:image" content="https://你的实际域名.com/img/栗子.png" />
<meta itemprop="image" content="https://你的实际域名.com/img/栗子.png" />
```

### 2. 准备分享图片

**当前使用：** `/img/栗子.png`

**微信分享图片要求：**
- 建议尺寸：1200x630 像素（或 2:1 比例）
- 格式：JPG 或 PNG
- 大小：< 5MB
- 必须是绝对路径（包含完整域名）

**可选：创建专门的分享图片**

如果想要更好的卡片效果，可以创建一张专门的分享图：
1. 尺寸 1200x630 或 800x400
2. 包含标题、照片和关键信息
3. 保存为 `/public/img/share-card.jpg`
4. 在 meta 标签中引用

## 四、微信分享限制

### 1. 普通链接分享

微信对**没有公众号**的普通链接支持有限：
- ✅ 会读取 title 和 description
- ✅ 可能显示图片（不稳定）
- ❌ 无法自定义分享图标
- ❌ 无法自定义分享文案

### 2. 微信公众号增强（可选）

如果有微信公众号，可以使用 JS-SDK：

```javascript
// 需要先在微信公众平台配置域名白名单
wx.config({
  // 配置参数
});

wx.ready(function() {
  wx.updateAppMessageShareData({
    title: '唐浚川·Lizi 百日邀请函',
    desc: '在爸爸妈妈的精心"翻炒"下，这颗小栗子已经"香甜软糯"100天啦！',
    link: 'https://your-domain.com/',
    imgUrl: 'https://your-domain.com/img/栗子.png'
  });
});
```

**需要：**
- 认证的微信公众号
- 在公众平台配置 JS 安全域名
- 后端生成签名（需要服务器）

   ```bash
   git add index.html
   git commit -m "配置微信分享卡片"
   git push
   ```

2. **等待部署完成**（2-3 分钟）

3. **清除微信缓存**
   - 微信会缓存链接信息
   - 部署后等待 10-30 分钟
   - 或使用新的查询参数（如 `?v=2`）强制刷新

4. **测试分享**
   - 在微信中打开链接
   - 分享给朋友查看效果

## 九、常见问题

**Q: 为什么我看不到分享卡片？**  
A: 
1. 检查域名是否正确
2. 图片 URL 是否可访问
3. 等待微信缓存更新（10-30 分钟）
4. 微信对个人网站支持有限，可能只显示标题

**Q: 如何自定义分享图标？**  
A: 需要微信公众号和 JS-SDK，普通链接无法完全自定义

**Q: 分享到朋友圈没有图片？**  
A: 朋友圈的显示规则更严格，建议：
1. 使用 1:1 的正方形图片
2. 图片大小 < 300KB
3. 确保 HTTPS 访问
